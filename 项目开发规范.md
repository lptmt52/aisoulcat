# aisoulcat 项目开发规范

## 1. 代码风格 (Code Style)
- 本项目继承了 VS Code 的代码风格。
- 使用 TypeScript 进行开发。
- 遵循 `.editorconfig` 和 ESLint 配置。
- 变量命名使用小驼峰式 (camelCase)，类型和类名使用大驼峰式 (PascalCase)。

## 2. Git 提交规范 (Commit Convention)
提交信息应遵循以下格式：
`<type>(<scope>): <subject>`

常用 type:
- `feat`: 新功能
- `fix`: 修补 bug
- `docs`: 文档变更
- `style`: 代码格式变更 (不影响代码运行的变动)
- `refactor`: 重构 (既不是新增功能，也不是修改 bug 的代码变动)
- `test`: 增加测试
- `chore`: 构建过程或辅助工具的变动

## 3. 开发流程 (Development Workflow)
- 所有新功能应当在独立的 feature 分支开发。
- 提交 PR 前需确保通过 Lint 检查和所有单元测试。

## 4. 技术栈 (Tech Stack)
- 核心：VS Code codebase (OSS)
- 语言：TypeScript
- 框架：Electron / React (部分 UI 层)
